    Programming Assignment

    To ensure that the example requests and responses show correctly, the language selected should be JavaScript fetch and the drop down on the example request should not be on default.

    The API has several methods to provide access to the JSON file through the REST API. A user can upload a new post, create a new account, delete a post or comment on a post which is done using a POST method. The user can view and search the posts and user profiles using a GET method. The entities I have used are people, pictures and comments.

    The methods access two JSON files posts.json which stores the details of the posts on the site and accoubts.json which stores the details of the user accounts.

    GET /pic
    http://127.0.0.1:8090/pic?name=User 4

    Sends the profile picture from the accounts.json file, corresponding to the user who is trying to log in. This is then displayed in navigation bar.

    PARAMS
    name
    User 4

    Recieves the URL for the user profile picture



    Example Request
    • Default
    • /pic
    var raw = "";

var requestOptions = {
  method: 'GET',
  body: raw,
  redirect: 'follow'
};

fetch("/pic", requestOptions)
  .then(response => response.text())
  .then(result => console.log(result))
  .catch(error => console.log('error', error));

    POST /newpost
    http://127.0.0.1:8090/newpost?title=New test title&image=www.testimage.com&des=Test description.&user=User 1

    Receives the title, image URL, descrtiption and user from the client side. Iterates through the exisiting posts and checks if the title of the new post matches any of the exisiting posts. There is authentication to ensure there is not a duplication of an exisiting post. A 403 error code will be returned if a post with the same title already exists. Otherwise, the new post will be written to the posts.json file.

    PARAMS
    title
    New test title

    The title of the new post

    image
    www.testimage.com

    The URL of the image associated with the post

    des
    Test description.

    The description of the post

    user
    User 1

    The name of the user who is currently logged in
